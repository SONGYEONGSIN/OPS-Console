## 1. 배경

`dashboard5.html`은 "운영실 OPSROOM" 서비스 운영 상황실 대시보드로, 종이 질감 배경 · 한글·영문 혼용 타이포그래피 · 주홍(vermilion) 낙관 배지를 특징으로 하는 에디토리얼 톤의 시각 정체성을 가진다. 현재 반응형 처리는 데스크탑 축소만 고려되어 있다:

- `@media (max-width: 1100px)` — 사이드바·인스펙터 폭 축소, 테이블 한 열 숨김
- `@media (max-width: 860px)` — 사이드바·인스펙터 완전 숨김

모바일(≤767px) 사용자는 타이틀바·메뉴바·탭 바·툴바가 그대로 노출되어 세로 공간 낭비가 크고, 콘텐츠 영역 외 인터랙션이 사실상 불가능하다. `body { overflow: hidden; height: 100vh }` 고정으로 좁은 화면에서 콘텐츠가 잘려도 스크롤되지 않는다.

## 5. 3단 레이아웃 → 드로어 전환

**데스크탑 DOM 구조는 그대로 유지.** 사이드바·인스펙터를 CSS `position: fixed` + `transform`으로 드로어화.

### 5.1 동작 규칙

| 뷰포트 | 사이드바 | 인스펙터 |
| --- | --- | --- |
| ≥ 1024px | grid 칸 상시 노출 | grid 칸 상시 노출 |
| 768–1023 | grid 칸 상시 노출 | `fixed; transform: translateX(100%)`, 토글 시 `.open` |
| ≤ 767 | `fixed; transform: translateX(-100%)`, 토글 시 `.open` | 동일 (우측) |

### 5.2 스크림

드로어 열릴 때 반투명 오버레이: `background: rgba(21, 18, 12, 0.35)` (ink 컬러 기반). 스크림 탭 시 닫힘.

### 5.3 열림 트랜지션

- 250ms `cubic-bezier(0.2, 0.0, 0.2, 1)` (ease-out)
- `transform` + 옅은 그림자 `box-shadow: -8px 0 24px rgba(21, 18, 12, 0.08)` (인스펙터는 좌측 그림자)
- `prefers-reduced-motion: reduce` 시 120ms로 단축

### 5.4 닫기 방법

1. 스크림 탭
2. 드로어 헤더 × 버튼
3. ESC 키
4. 스와이프 (사이드바: 왼쪽으로, 인스펙터: 오른쪽으로; **드로어 폭의 50% 초과** 이동 시 닫기, 아니면 snap-back)

드로어 폭:
- 사이드바: `min(84vw, 300px)` (모바일에서)
- 인스펙터: `min(92vw, 360px)` (모바일에서, § 8 참조)

### 5.5 접근성

- 드로어 요소: `role="dialog"`, `aria-modal="true"`, `aria-labelledby` 헤더 연결
- 열림 시 focus trap (드로어 내부로 tab 순환)
- 열림 시 바깥 영역 `inert` 속성 (Chromium/WebKit 지원) + `body { overflow: hidden }` 스크롤 락

## 10. CSS 토큰 체계

기존 팔레트 변수(washi/ink/vermilion 등)는 유지. 타이포·간격·터치 스케일 변수를 추가한다.

```css
:root {
  /* 타이포 */
  --text-xs:  11px;
  --text-sm:  12px;
  --text-md:  13px;  /* 본문 기준 */
  --text-lg:  15px;
  --text-xl:  20px;
  --text-2xl: 26px;

  /* 간격 */
  --space-1: 4px;
  --space-2: 8px;
  --space-3: 12px;
  --space-4: 16px;
  --space-5: 20px;
  --space-6: 28px;

  /* 터치 */
  --tap-min: 36px;
}

@media (max-width: 767px) {
  :root {
    --text-md: 14px;
    --text-xl: 18px;
    --text-2xl: 22px;
    --tap-min: 44px;
  }
}
```

### 10.1 적용 원칙

- 본문·메타 글꼴은 모두 토큰으로 치환
- `h1` (컨텐츠 헤드)은 `clamp(22px, 4.5vw, 26px)`
- 사이드바·인스펙터 필드 패딩은 `--space-*` 사용
- 버튼·탭·사이드바 항목 `min-height: var(--tap-min)`
- Pretendard `letter-spacing`·`font-feature-settings` 유지
- 언어 라벨 장식(`.label-en`, `.kr` 클래스)은 모바일에서도 유지

### 10.2 장식 톤 조정

- washi 노이즈 오버레이: `≤767px`에서 `opacity: 0.3` (데스크탑 0.5 → 모바일 0.3). 작은 화면에서 노이즈가 텍스트 가독성 방해.
- `font-variant-numeric: tabular-nums`는 유지 (ID·타임스탬프 정렬).

## 11. DOM 추가

모바일 전용 요소를 HTML에 추가한다. 데스크탑에서는 `display: none`.

```html
<header class="appbar" role="banner">
  <button class="btn-hamburger" aria-label="메뉴 열기" aria-expanded="false"
          aria-controls="sidebar">☰</button>
  <div class="appbar-title">운영실 <em>·</em> OPSROOM</div>
  <button class="btn-notif" aria-label="알림 3건">
    ✉<sup>3</sup>
  </button>
  <span class="appbar-led" aria-label="연결됨" role="status"></span>
  <button class="btn-inspector" aria-label="상세 보기" aria-expanded="false"
          aria-controls="inspector" disabled>상세 ▸</button>
</header>

<div class="scrim" aria-hidden="true" data-drawer-scrim></div>
```

사이드바(`<aside class="sidebar" id="sidebar">`) · 인스펙터(`<aside class="inspector" id="inspector">`)에는:
- `role="dialog"` (드로어 상태에서)
- `aria-modal="true"` (드로어 상태에서)
- `aria-labelledby` 헤더 id 연결
- 닫기 `<button class="drawer-close" aria-label="닫기">×</button>`

```js
// 4. 스크롤 락
//    - 드로어 열림 시 body { overflow: hidden }
//    - 닫힘 시 이전 overflow 값으로 복원
//
// 5. 서비스 행 탭 → 인스펙터 자동 열기 (모바일 한정)
//    - matchMedia('(max-width: 767px)').matches일 때만
//    - .doc-row 클릭 시 [상세 ▸] 버튼 활성화 + 드로어 자동 open
//
// 6. 터치 스와이프 닫기
//    - pointerdown / pointermove / pointerup
//    - 사이드바: 왼쪽 방향 이동 > 드로어 폭의 50% → 닫기, 아니면 snap-back
//    - 인스펙터: 오른쪽 방향으로 동일 (드로어 폭 기준)
//    - 스와이프 중에는 transform transition 비활성화 (즉시 반응)
//
// 7. 인스펙터 아코디언 (모바일 한정)
//    - .ins-section 헤더 클릭 시 .collapsed 토글
//    - 기본값: 실시간 지표 / 담당·온콜 / 활동 기록 펼침
```

## 14. 호환성 & 접근성

- 브라우저: iOS Safari 14+, Chrome Android 90+, 데스크탑 최신 2버전
- `prefers-reduced-motion: reduce` 시 트랜지션 120ms로 축소
- 모든 인터랙티브 요소 `min-height: var(--tap-min)` 보장
- 드로어에 `role="dialog"`, `aria-modal="true"`, `aria-labelledby`
- 포커스 트랩 + 닫을 때 focus 복귀
- 색상 대비: 기존 washi 팔레트의 `--ink` / `--muted` 대비 유지 (WCAG AA 수준)

## 15. 검증 체크리스트

### 레이아웃 회귀
- [ ] 1440px — 현재 데스크탑과 시각적으로 동일
- [ ] 1280px — 현재와 동일
- [ ] 1024px — 사이드바·인스펙터 축소 (기존 1100 규칙과 유사)
- [ ] 768px — 인스펙터 드로어, 사이드바는 상시 노출
- [ ] 480px — 모바일 앱바 등장, 양쪽 드로어
- [ ] 360px — 콤팩트 모드 정상

- [ ] [상세 ▸] → 인스펙터 드로어 열림/닫힘 (서비스 미선택 시 비활성화)

- [ ] 스와이프로 드로어 닫힘 (사이드바 ← , 인스펙터 →)

## 16. 비구현 사항 (YAGNI)

- 다크 모드 (기존 테마 변수에 이미 일부 있으나 전환 미구현)
- 실시간 데이터 바인딩 (전부 정적 HTML)
- 토폴로지 / 카드 뷰 전환 (툴바에 버튼만 존재, 구현 없음)
- 푸시 알림 / 서비스 워커
- 다른 대시보드(4, 6~9) 적용 (패턴은 재사용 가능하도록 토큰화했지만 개별 파일은 이번 작업 대상 아님)
