# 경위서(Incident Explanation Report) 메뉴 — 설계 노트 (WIP)

> 상태: **설계 단계 (brainstorm)** — 샘플 양식 대기 중, 코드 구현 전.
> 작성일: 2026-06-01

## 목표

자료보관 메뉴 아래 "경위서" 화면. 운영자가 경위서 양식을 화면에서 바로 작성하고,
팀장 승인을 거쳐 고객에게 PDF로 메일 발송하며, 생성된 Word 파일을 SharePoint
`운영부 > 06. 경위서` 폴더에 업로드하는 일련의 흐름을 OPS Console 안에서 처리한다.

## 결정된 사항 (사용자 확정)

| 항목 | 결정 |
|------|------|
| 도메인 구조 | **독립 도메인**(자료보관 > 경위서)이되, 기존 incidents(사고관리)에 **선택적 연결** 가능 (optional FK) |
| 승인 흐름 | **앱 내 팀장 승인** — 팀장이 OPS Console 로그인하여 대기 목록에서 승인 |
| 출력 형식 | **Word + PDF 둘 다** — .docx(SharePoint 업로드용) + PDF(고객 메일 첨부용) |
| SharePoint 업로드 | 생성된 .docx를 `06. 경위서` 폴더에 업로드, **로그인 운영자 이름으로** |

### 흐름(잠정 status machine)

```
draft → pending_approval → approved → sent
            (팀장 승인)      (고객 메일 발송 + SharePoint 업로드)
```

## 재사용 가능한 기존 패턴 (조사 완료)

- **PDF 생성**: `src/lib/pdf/handover-pdf.tsx` / `backup-request-pdf.tsx` — @react-pdf/renderer + Pretendard, `renderToBuffer()` → 메일 첨부
- **메일 발송**: `src/lib/microsoft/sendmail.ts` `sendGraphMail` — 로그인 운영자 본인 메일박스 발송, PDF 첨부 지원
- **승인/2인 확인 흐름**: `src/features/handover/progress-*.ts` — draft→진행→확인 status 전이 패턴
- **list-variants 신규 도메인 등록**: `registry.ts` 1줄 + `types.ts` Variant union 1줄 + View/EditForm/Table/filters.ts
- **부서별 보고자 자동 매핑**: `src/features/incidents/` REPORTER_BY_DEPARTMENT 패턴
- **SharePoint env**: `src/app/dashboard/settings/_env.ts` SHAREPOINT_*, Graph auth `src/lib/microsoft/auth.ts`

## 새로 만들어야 하는 것

- **경위서 전용 필드/섹션** — 샘플 양식 받으면 확정 (⏳ 대기)
- **.docx 생성** — 코드베이스에 docx 라이브러리 없음 → `docx` npm 신규 추가 필요
- **SharePoint 업로드(PUT)** — 현재 SharePoint 연동은 전부 읽기 전용(GET). Graph `/drives/{driveId}/items/{path}:/content` PUT 신규 구현 필요. `06. 경위서` 폴더의 driveId/folder 경로 env 추가 필요
- **경위서 페이지** `src/app/dashboard/incident-reports/` (또는 자료보관 하위) + 메뉴 등록(`_data` sidebar)
- **경위서 메일 템플릿** (handover/incidents 스타일 참조)
- **팀장 승인 권한 모델** — 누가 승인자인지 결정 로직

## 미결정 (다음 세션에서 확정할 것)

1. **경위서 양식 필드/섹션 구조** — `docs/refs/`에 샘플 양식 파일 넣으면 읽고 확정 (가장 중요)
2. **승인자(팀장) 지정 방식** — 운영자 소속 팀 기반 / 경위서별 지정 승인자 / admin 중 택1
3. **SharePoint 업로드 "로그인 계정 이름"의 의미** — 파일명·폴더에 운영자 이름 삽입(간단) vs 실제 SharePoint '수정한 사람'을 운영자로(위임 인증 필요, 복잡)
4. **고객 메일 주소 출처** — incidents 연결 시 자동 / 수동 입력
