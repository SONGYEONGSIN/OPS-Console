# Plan: /dashboard/my-todo 2 탭 재설계 — Weekly Planner + Project Gantt

## Goal

`/dashboard/my-todo`를 URL 기반 2 탭(`?tab=weekly|project`, default weekly)으로 재설계.

- **Tab 1 (원서접수 — weekly)**: 본인 todos를 카테고리·우선순위·상태·진행률 메타와 함께 주(week) 그리드 + 테이블로 표시
- **Tab 2 (프로젝트)**: 본인 `projects`(부모) + 1단계 sub-task(`project_tasks`)를 담당자·시작·마감·우선순위·진행률 + Gantt 차트(월별 timeline 막대)로 표시

기존 services 기반 D-60 planner 흐름은 완전 폐기. RLS는 본인 only.

## Approach

brainstorm 대안 A 채택:
- `todos` 컬럼 확장 (nullable `category`/`progress`/`status` 추가, `source_service_id` FK 제거)
- 신규 도메인 `projects` + `project_tasks` (1:N, sub-task 깊이 1단계)
- Gantt는 라이브러리 없이 Tailwind grid + 토큰 직접 구현
- list-variants 패턴 활용 — `weekly-todo` / `project` / `project-task` 3 variant 추가
- 기존 `MyTodoLayout.tsx` 등 services-based planner 자산 삭제

## PR 전략

**단일 PR** (사용자 결정 2026-05-19). 42 파일 변경을 1 branch에서 완성 후 `feat/my-todo-redesign` → main squash merge.

## 사전 검증 결과

- ✅ **source_service_id 데이터 손실 리스크 해소**: production DB 조회 결과 `source_service_id NOT NULL` row **0건** (전체 todos 3건). drop column 안전.

## Out of Scope

- sub-task 깊이 2단계 이상 (v1 1단계)
- 카테고리 admin 마스터 enum (v1 자유 입력 string)
- 프로젝트 공유 (다른 사용자 담당자 지정) — 본인 only
- Gantt 주/월 단위 zoom (v1 일 단위만)
- todos.source_service_id 데이터 보존 (사용 0건 확정)
- sub-task 진행률 합산을 parent 자동 반영 (수동 입력 유지, 보조 표시는 v1에 포함)
