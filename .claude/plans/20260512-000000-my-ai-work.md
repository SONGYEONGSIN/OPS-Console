# Plan: 내 작업(my-ai-work) — AI 활용 등록 메뉴

## Goal

운영자가 본인의 AI 활용 작업 결과물(요약 · 재사용 프롬프트 · 결과물 링크 · 절감 시간)을 등록·열람·편집할 수 있는 페이지를 추가한다. 사이드바 `분석 · AI > AI & 자동화 > 내 작업`(slug `my-ai-work`)에 이미 등록된 항목을 실제 페이지로 연결. 모든 운영자에게 공개되어 재사용 프롬프트가 동료 업무에 즉시 복사되어 사용되는 것이 핵심 가치.

## Approach

대안 A 채택. 운영부 전체 공개 RLS(SELECT 전원, INSERT/UPDATE/DELETE 본인만) + ListPattern default variant 재활용(`columns` prop 신설)으로 9개째 variant 부채를 회피 + 작업 일자는 자유 입력(zod cutoff 없음). DB 연동이 필수이므로 동적 `[slug]` 라우트가 아니라 `src/app/dashboard/my-ai-work/page.tsx` 정적 라우트로 분리(my-todo / receivables / feedback 패턴 따름).

기존 검증된 패턴 재사용:
- 마이그레이션 분할: `<date>_ai_work_table.sql` + `<date>b_ai_work_rls.sql` (receivables_mail_sends 패턴)
- features/ai-work/{schemas,actions,queries}.ts (posts/todos 패턴)
- ListPattern default variant + columns prop으로 컬럼 헤더·셀 렌더만 주입 (variant 폭증 회피)
- 인스펙터는 InspectorListBody에 ai-work 분기 추가 (markdown 렌더 + 복사 버튼)

## Out of Scope

- 3단계 권한(private/team/public) 분기 — 운영부 공개 단일 정책
- 작업 일자 cutoff 검증 — 자유 입력
- 사이드바 count "5" 하드코드 동적화 — 사이드바 구조 전반 영향이라 follow-up PR로 분리
- ListPattern 9개째 variant 추출 — 별도 리팩토링 PR (brainstorm 명시)
- 재사용 프롬프트 "복사 클릭" 정량 계측 — follow-up
- DESIGN.md 신규 토큰 추가 — 기존 토큰(vermilion / ink / muted / washi-raised / line-soft) 으로 충분
