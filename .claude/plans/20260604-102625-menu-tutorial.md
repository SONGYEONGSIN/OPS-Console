# Plan: 메뉴별 상세 튜토리얼

## Goal
정적 5스텝 전역 투어(driver.js)를 **메뉴(slug)별 상세 워크스루**로 확장한다. 권한으로 필터된 `sections`를 소비해 각 메뉴마다 ①개요 ②"리스트 행 클릭 → 오른쪽 인스펙터가 열린다"는 인터랙션 안내 ③핵심 버튼 기능 설명 3종 스텝을 동적 생성한다. 전체 메뉴 대상.

## Approach
확정 대안 A — 인스펙터 자동 조작 없이 driver.js 스포트라이트 + 설명만 사용. `tutorial-menu-copy.ts`에 slug별 `{overview, interaction, buttons[]}` 단일 사전, `build-menu-steps.ts`에 권한 필터된 `sections`를 소비하는 순수 함수 빌더(누락 slug skip). 정적 `TUTORIAL_STEPS`는 `INTRO_STEPS`(전역 인트로)로 보존하고 메뉴 동적 스텝과 결합. 사이드바에 `data-tutorial-slug` 앵커, TutorialTour에 온디맨드 '가이드' 버튼. seen 키 `v1→v2`. 인프라 1 PR → 콘텐츠 batch 여러 PR로 분리.

## Out of Scope
- 인스펙터 자동 클릭/시연(대안 B) — 빈 목록·데이터 변동·상태오염 위험으로 제외.
- 누락 slug에 대한 generic 임시 문구(폴백 금지, `donts.md`) — skip + dev 경고로 처리.
- 정적 매뉴얼/도움말 드로어(대안 C) — 추후 버튼 세부 많은 메뉴의 하이브리드 여지로만 남김.
- 버튼별 실 DOM 셀렉터 앵커(인스펙터 미오픈 전제) — 안정 앵커 도입은 후속 선택지.
