# Plan: `/dashboard/reports` 분석보고서 메뉴

## Goal

운영부의 누적 데이터(services/incidents/contracts/receivables/handover_progress/backup_requests/mail_sends/worklog 8 source)를 **기간별 KPI 8 카드 대시보드**로 노출. 동일 화면 하단에 **저장된 리포트 목록 placeholder**. 기간 selector 5 옵션(이번 주/이번 달/지난 달/분기/연간) + 전 기간 대비 증감 표시.

## Approach

settings/operating-guide/manuals 톤 일관 — `PageHeader(autoRefresh)` + `<section p-5..p-7>` + 자체 컴포넌트.

**Data flow**:
1. URL `?period=this-month` (기본) — 5 옵션 enum
2. Server page에서 `getReportKpis(period)` 호출 → 8 항목 병렬 측정 + 전 기간 비교
3. 클라이언트 컴포넌트(`PeriodSelector`)는 selector 토글만 (URL push)
4. `KpiCard` 8회 렌더 — 라벨/현재값/증감/단위

**1차 MVP**:
- KPI 8 카드: 실측
- 리포트 목록: 2~3 정적 mock entries + "+ 새 리포트" 버튼 disabled (placeholder)

## Out of Scope (follow-up)

- 리포트 자동 생성·저장·PDF 출력
- 리포트 상세 페이지
- Sparkline 차트 (현재는 숫자 + 증감 화살표만)
- Custom 기간 선택 (5 고정 옵션만)
- KPI 카드 클릭 → 도메인 페이지 deep link (1차는 read-only display)

## 리스크

1. **SharePoint contracts/receivables fetch 비용** — 전체 시트 fetch 후 client filter. 60s cache 권장. 페이지 진입마다 호출되면 부하 → 캐싱 또는 next-fetch revalidate 활용
2. **period 경계 계산 오차** — KST 기준 자정, 분기 시작일 등. `Intl.DateTimeFormat` + 명시적 +09:00 timezone 처리
3. **전 기간 비교 분모 0** — 지난 달이 0건이면 % 계산 불가. `null` 또는 "신규" 표시 처리
4. **메뉴 노출 권한** — viewer도 통계 read 가능해야 함. requireMenu에서 막지 않도록 사이드바 권한 그대로
