# Plan: `/dashboard/manuals` 메뉴 — SharePoint 운영부 매뉴얼 폴더 read-only 뷰

## Goal

운영부가 SharePoint 메인 드라이브 `운영부/05. 매뉴얼` 폴더에 보관하는 매뉴얼 65+ 파일 + 4 하위 폴더(총 89 children)를 `/dashboard/manuals` 메뉴에서 ListPattern + Inspector로 조회한다. 행 클릭 시 SharePoint 웹 URL을 새 탭으로 연다. 하위 폴더는 query param 드릴다운으로 children 노출. 편집·다운로드는 SharePoint UI로 위임 — Folio는 read-only 카탈로그.

## Approach

**대안 A 채택** (brainstorm spec): 접두사 그룹(A. / B. / C. … I.)으로 카테고리 자동 분류 + ListPattern 기본 검색 + Inspector 메타 view. 다른 16 variant와 일관, 별도 트리 UI 없음.

**Data flow**:
1. `listManualChildren({ parentItemId? })` → Graph `GET /drives/{drive}/items/{item}/children?$top=200` (`SHAREPOINT_MANUAL_ITEM_ID` 기본, 하위 폴더는 query param `itemId`).
2. children → `ManualRow[]` 매핑 (folder vs file, 이름에서 접두사 추출, webUrl/lastModifiedDateTime/size/mimeType).
3. SSR 페이지가 ListPattern variant=`manual` 로 렌더, `inlineFilters`에 카테고리 chip 또는 `parentItemId` breadcrumb.
4. Row click → folder면 `?itemId=...&path=...` 푸시 (서버 재페치), file이면 `window.open(webUrl, "_blank")` (Inspector View에 표시되는 링크 버튼).

**workbook-session 사용 안 함** — 매뉴얼은 Excel sheet API가 아닌 driveItem children API이므로 401 retry는 `getGraphToken` cache invalidate 후 1회 재시도만 (contracts 패턴 그대로 차용, workbook-session 부분만 제외).

## Out of Scope

- 매뉴얼 편집/업로드/삭제 — SharePoint UI 위임
- 다운로드 endpoint — 행 클릭 시 SharePoint 웹으로 열기 (브라우저 미리보기 + 다운로드)
- 사이드바 카운트 배지 (`menu-counts` 통합)
- 검색 인덱싱 / 본문 fulltext — ListPattern 기본 파일명 검색만
- 권한 분리 — 운영부 전원 read 가능 (현재 메뉴 가드 통과 = 노출)
- 재귀 트리 UI / breadcrumb 자동 생성 — 1단계 드릴다운만 (운영부 폴더 깊이 ≤ 2)
- 양방향 sync (SharePoint ↔ Folio)

## 리스크

- **카테고리 접두사 가정 위반**: 일부 파일이 `A.` 형식이 아닐 수 있음 (예: "README.md", "임시-2026.docx"). → `extractCategoryFromName`이 `null` 반환 시 "기타" chip으로 fallback 처리 명시 (filters에 "기타" 추가하거나 chip 미적용 행은 모든 chip에서 노출). T3 테스트 케이스에 명시 필요
- **89 items 페이징**: Graph `$top=200`이면 1 페이지로 충분하지만 향후 증가 대비 `@odata.nextLink` 무시 → 200 초과 시 누락. T4 주석에 명시, 100 도달 시 follow-up 이슈
- **하위 폴더 라우팅 보안**: `itemId` 검증 없으면 임의 driveItem 노출 위험. → SharePoint이 같은 drive 내 권한 일관 적용하므로 driveId 고정 + itemId만 query로 받는 패턴은 안전. 단 응답에서 `parentReference.driveId` 검증 1단 추가 권장 (T4)
- **registry satisfies 컴파일 에러 윈도우**: T6에서 union 추가 후 T9 등록 전까지 typecheck 실패 상태 — T6→T9는 같은 commit에 묶거나 빠른 sequence로 진행
- **사이드바 slug 변경 영향**: 기존 `slug:"manual"`을 사용 중인 다른 곳(예: bookmark, breadcrumb 테스트)에 영향 가능 → T11 전 `rg "slug.*manual[^s]" src/` 로 사전 영향 확인 필수
