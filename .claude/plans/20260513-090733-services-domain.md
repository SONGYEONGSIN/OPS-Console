# Plan: 전체 서비스 메뉴 — 대학 입시 원서접수 서비스 카탈로그 (PR-1)

## Goal

- `/dashboard/services` 정적 라우트 신설 — 2511행 입시 원서접수 서비스 카탈로그를 본인 필터 + 컬럼 필터 + 대학명·서비스명 검색과 함께 운영
- list-variants 신아키텍처에 4번째 도메인 추가 (ai-work / cohort / backup / services)로 패턴 안정성 검증
- Google Sheets → CSV → Folio 일회성 import 스크립트 (운영자 매칭 + 카테고리 Excel 보강) 동봉
- 사이드바 `services` slug (count "179" 하드코드) 기 등록 상태 — 정적 라우트로 dynamic [slug] 오버라이드

## Approach

대안 A 채택 (brainstorm 결정). 일회성 Sheets→DB import 후 Folio가 source-of-truth. operators 마스터와 RLS 연동된 본인 담당 필터가 Folio의 차별 가치.

핵심 패턴:
- **마이그레이션 2 파일**: 테이블 (20 컬럼, solo 포함) + RLS (ai_work 패턴 일관 — 운영자 전원 read/write)
- **features 3 파일**: schemas/queries/actions + tests 3 (RED 선행)
- **list-variants/services 4 슬롯**: View/EditForm/Table/filters + registry 1줄 + types Variant union 1줄
- **page**: 정적 라우트 — RSC에서 listServices + ownerByEmail 매핑 → ListRow 변환, search/filter params 처리
- **import script**: `scripts/services-import.mjs` — CSV 파싱 + Excel category 보강 + operators name→email best-effort 매칭 + service_id UNIQUE upsert
- **검색·필터 UX**: ilike 1차 (200ms debounce), to_tsvector + gin은 응답 > 500ms 측정 시 follow-up

HARD-GATE: ~17~20 파일 → **간략 설계 등급**

## 리스크

- **운영자/개발자 name → email 매칭 실패** — Excel 운영자 text가 operators 마스터에 없거나 동명이인일 때. 1차 mitigation: text fallback + `unmatched_operators` 로그. EditForm에서 admin이 수동으로 operators select로 재지정 가능. 후속에서 admin 매칭 정정 UI 검토.
- **service_id 충돌** — 외부 PIMS 7자리와 Folio 자체 생성 ID. mitigation: service_id는 항상 user input (PIMS 부여값 보존), Folio 자체 생성도 사용자 직접 입력. zod에서 bigint 검증 + DB UNIQUE 제약으로 충돌 거부. 23505 → "이미 존재하는 service_id" 한국어 메시지 매핑.
- **대용량(2511행) 검색·페이지네이션 응답 시간** — 1차 ilike + 인덱스 6종. 측정: `EXPLAIN ANALYZE` + 응답 시간 로그. 500ms 초과 시 follow-up에서 to_tsvector + gin 도입 (Out of Scope).
- **카테고리 enum 12개 — 운영부의 신규 카테고리 추가 부담** — check 제약 vs text 자유. 결정: check 제약 유지 (data 정합성 우선). 신규 카테고리는 마이그레이션 1줄로 추가.
- **imported_at vs created_at 의미 분리** — import row는 둘 다 채워짐, folio_create row는 imported_at null. import 스크립트에서 `imported_at: new Date().toISOString()` 명시 + folio_create는 default null.
- **Google Sheets CSV 파싱** — quote / 한글 / 빈 셀. mitigation: `csv-parse` 옵션 `{ bom: true, trim: true, skip_empty_lines: true }` + 파싱 후 빈 문자열 → null 변환.
- **listOperators name 매칭 시 trim/normalize** — 시트 운영자 컬럼이 공백 포함 가능. mitigation: `.trim().normalize('NFC')` 양쪽 적용.
- **EditForm operators select 데이터 주입** — 21+22명을 props로 전달. page.tsx에서 `OPERATORS` import 후 props로 전달. EditFormProps 확장 1필드.
- **RLS 정책 회귀** — mitigation: 마이그레이션 적용 후 `select has_table_privilege('authenticated','public.services','INSERT')` 검증.
- **ListRow 필드 17개 추가로 인한 타입 비대** — 이미 backup/receivables로 ListRow 비대. mitigation: 본 PR optional 추가만, 후속에서 도메인별 sub-type 분리 epic 검토 (Out of Scope).
- **tdd-enforce hook strict 모드** — types/registry는 type-level 면제 (tdd.md 예외), page.tsx는 T16 수동 smoke로 대체.
