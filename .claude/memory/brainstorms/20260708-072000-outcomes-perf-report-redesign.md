# Brainstorm: 성과리포트 재설계 — 관리자 중심 AI 성과 측정 체계

## 의도
- **무엇을**: 관리자 중심 성과리포트. 개인목표 + 실행계획 → 성과지표 80%(정량 자동집계 + 목표달성) + 관리자지표 20%(정성 루브릭) → 월간·분기·반기 리포트 발행. 산출물: 성과 측정 가이드라인 / 리포팅 템플릿 / 실적용 리포트.
- **누가**: 관리자(admin)가 팀원 평가·채점 + 리포트 발행. 팀원은 본인 목표·실행계획·성과지표(가중치 배분) 작성 + 결과 열람.
- **왜 지금**: 상반기 지표·템플릿 설계 → 반기 내 1회 이상 실적용 (일정 확정). AX 과제 성과관리의 표준화 필요.
- **성공(KPI)**: 표준 지표 N개 확정 + 측정주기·리포팅 템플릿 완비 + 실적용 리포트 최소 1회 발행.

## 제약
- **기술/코드베이스**: 기존 8단계 워크플로우 + 5테이블(cycles/assignments/goals/plans/reviews)은 backbone만 있고 UI↔액션 미연결·데이터 0건 → 재구성 자유도 높음. admin 전용(RLS admin 분기 존재). ListPattern + 색상토큰 표준 준수. 기존 버그: actions revalidate 경로 `/dashboard/performance`(실제 `/dashboard/outcomes`).
- **데이터**: `closing_services`는 `operator_name`(Moa 표기)만 → operators 이메일 매핑 필요. before/after 시간절감·오류율은 원천 데이터 없음 → 수동 입력 불가피.
- **비즈니스**: 반기 일정, admin 주도, 팀원 자율 가중치 배분.

## 정량 데이터 소스 (자동 집계 가능, 운영자 귀속 확인됨)
- 서비스마감 `closing_services.operator_name` — 담당 서비스·마감 완수
- 사고보고 `incidents.assignee_email,status` — 담당 건수·처리완료율·무사고
- AI자동화 `ai_work.author_email` — 팀원별 AI 결과물 수
- 계약체결 (contracts 시트 operator) — 담당 계약 완료 건수 (기존 집계 재사용)
- 할일 `todos.assignee_email,done_at,due_at` — 완료율·기한 준수
- 인수인계 `handover_progress` / 백업 `backup_requests` / 미수 회수 / worklog(참고)
- → before/after 시간절감·오류율만 수동 입력.

## 추천 + 근거
**대안 B (관리자 중심 4단계 단순화)** — 사용자 결정 확정.
- 근거: "관리자 중심" 명시, 8단계는 팀원↔평가자 왕복이 많아 과설계. 정량 자동집계 패널이 핵심 차별점.
- 기각 A: 공식 절차가 엄격해야 할 때만 가치. 현 조직 규모엔 오버.

### 확정된 설계 결정 (사용자 답변)
1. **워크플로우 4단계**: ①목표설정(팀원) ②실행계획+성과지표 정의·가중치 배분(팀원) ③정량 자동집계 + 관리자 루브릭 채점(시스템+관리자) ④리포트 발행(관리자)
2. **정량 집계 = 자동+수동 혼합**: 건수·완수율은 시스템 자동, before/after 시간절감·오류율은 수동 입력
3. **가중치 = 팀원이 영역별 자율 배분, 성과지표 합 = 80%** (검증 규칙), 관리자지표 = 20% 고정
4. **리포팅 주기 = 월간(정량 스냅샷) + 분기(집계) + 반기(최종 평가·발행)**
5. **관리자 채점 = 루브릭 척도(1~5) + 근거 코멘트 필수 + 정량 보조 패널 동시 표출**, 자기평가→관리자평가 2단계, 최종 = 성과80%+관리자20% 가중합 → 등급 자동

### 화면 구성 (admin 중심)
- 목록: 사이클×팀원 assignment + admin 단계 분포 요약
- 인스펙터(팀원 상세):
  - **개인목표** 섹션 (목표 + 세부)
  - **성과지표 80%** 섹션 — 지표 카드(활동지표) + 가중치(합 80% 검증) + 정량 자동집계 값 인라인 표출 + before/after 수동 입력
  - **관리자지표 20%** 섹션 — 태도·문화/협업/문제해결 루브릭 척도 + 코멘트
  - **종합** — 점수·등급 자동 + 리포트 발행(인쇄 페이지 재사용)

### 데이터 모델 변경 (예상)
- 재사용: goals=개인목표, plans=실행계획
- 신규/재정의: 성과지표(지표명·가중치·정량소스매핑·before/after·실적) / 관리자 루브릭 점수(항목별 척도+코멘트) / 월간 스냅샷 / 정량 집계 aggregator(소스별)
- 버그 수정 포함: revalidate 경로

### 1차 MVP 정량 소스 (확정)
- **3종: 서비스마감 + 사고보고 + AI자동화(내작업)** — 개인목표 3축(서비스 안정운영/무사고/AI 과제)에 1:1 대응, 마찰 모두 낮음
- 매핑: 서비스마감 `operator_name` → `operators.ts`(21명) name↔email 정적맵. 사고·AI작업은 email 직결
- aggregator = **레지스트리 패턴** (1 소스 = `operator_email + 기간 → 지표` 모듈 1개). 2차 확장 = 모듈 1개 + 등록 1줄
- **2차**: 할일(todos) → 계약체결 / **보류**: 인수인계·백업·미수·worklog
