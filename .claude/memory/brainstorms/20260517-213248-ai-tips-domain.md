# Brainstorm: TIP 공유 (ai-tips) 도메인 신설

작성: 2026-05-17 / 사용자: 송영석

영감: 사이드바 '분석·AI > AI & 자동화 > TIP 공유' 메뉴(slug `ai-tips`)가 등록만 되고 화면 미구현. my-ai-work와 별개로 운영부 공통 AI 팁 자산화.

## 의도

- **산출물**:
  - `ai_tips` 테이블 + RLS (my-ai-work 패턴 mirror)
  - `features/ai-tips/` (schemas / queries / actions / tests)
  - `_components/inspector/list-variants/ai-tips/` (View / EditForm / Table / filters)
  - `/dashboard/ai-tips/page.tsx` (list 패턴, ScopeChips, 검색·필터)
  - menu-counts 등록 (사이드바 동적 카운트)
- **사용자**:
  - 신입(viewer): TIP 둘러보면서 학습. reuse_prompt 1-click 복사로 즉시 활용
  - member: AI 도구 쓰다가 좋은 프롬프트 발견 시 등록. 본인 글 수정
  - admin: 모든 TIP 수정/삭제
- **트리거**: 사용자가 화면 구성 직접 요청. 사이드바 약속 이행 + AI 활용 자산화
- **성공 기준**:
  1. TIP 등록/조회 정상 동작
  2. reuse_prompt 1-click 복사 (AiWorkView 패턴 재사용)
  3. ai_tool / category 필터 + 본인/전체 ScopeChips
  4. 단위 테스트 + lint/typecheck 통과

## 제약

- **기술**:
  - 기존 list-variant open/closed 아키텍처 준수 (registry.ts 1줄 추가, dispatcher 무변경)
  - RLS: my-ai-work 동일 — authenticated read all / 본인 글만 update·delete / admin 모든 글 update·delete
  - reuse_prompt NOT NULL (TIP의 핵심 가치)
- **비즈니스**:
  - my-ai-work와 명확히 분리. TIP = 단편 + 즉시 재사용, my-ai-work = 사례 회고(긴 글)
  - 추천/좋아요/북마크는 후속 epic
  - 댓글/스레드도 후속 (Notion 위키로 충분히 대체)
- **코드베이스**:
  - ai_tool / category enum은 ai-work/schemas.ts에서 import 재사용 (중복 정의 X)
  - PostgREST cache reload `notify pgrst, 'reload schema'` 적용

## 추천 + 근거

**대안 A — 완전 새 도메인** (사용자 결정 2026-05-17)

근거:
1. **데이터 모델 자체가 다름**: my-ai-work는 사례 회고(시점·효과 측정), ai_tips는 즉시 재사용 자산(시점 무관)
2. **재사용성 우선**: reuse_prompt가 TIP의 핵심 → NOT NULL. my-ai-work에서는 옵션
3. **패턴 mirror로 비용 낮음**: 마이그/RLS/list-variant 모두 기존 패턴 그대로 복제 → 학습/리뷰 비용 ↓

기각 B: my-ai-work 스키마에 TIP 의미 컬럼이 추가되면 한 테이블이 두 컨셉을 담아 인지 부담 ↑. featured flag로는 사용자에게 "단편 vs 사례" 차이가 UI에서 안 보임. 전환 가치 있는 시점: 만약 TIP 등록량이 매우 적어 단독 테이블이 과잉이라 판단되면 features view로 전환 (가역).
