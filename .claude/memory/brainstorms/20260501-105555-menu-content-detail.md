# Brainstorm: 메뉴별 전체 콘텐츠 페이지 디테일 구성

## 의도

- **산출물**: 47 페이지 각각에 의미 있는 도메인-맞춤 콘텐츠/UI 디테일을 디자인 규칙 준수하에 채워 넣음.
- **사용자**: 송영석 (운영부 빌더). 시점: 시연/내부 피드백 + Supabase 연결 전 mock 운영 시뮬레이션.
- **트리거**: IA PR(#1) 머지 후 콘텐츠 빈약 지적, "메뉴 하나하나 디테일하게" 지시.
- **성공 기준**:
  1. 각 페이지 시연 가능한 도메인 맥락
  2. design tokens + Tailwind만 사용 (하드코딩 색상 0)
  3. 3회+ 반복 UI는 common 컴포넌트 추출
  4. 페이지당 단위 테스트 1개+

## 제약

- 기술: Tailwind v4 + design tokens, Next.js 16 prerender 호환
- 비즈니스: Supabase는 별도 epic. 페이지당 30~60분 추정, 분할 배포 필수
- 코드베이스: 5 패턴 컴포넌트 셸 그대로, `_data/patterns.ts` mock만 변경
