## 제약

### 기술
- Next.js App Router + Supabase + zod
- list-variants 신아키텍처 활용 (services variant 신설 — 1폴더+1줄)

## 추천 + 근거

**대안 A 채택**.

### 선택 근거
1. **사용자 명시 의도 부합** — "알려준 서비스만 등록하고, 추가된 서비스는 새로 추가하는 방향" = 일회성 import + Folio가 source
2. **운영자 가치 즉시** — 본인 필터 + 검색은 Sheets로는 어색. 조직 권한 메뉴 연동된 view는 Folio만의 가치
3. **서비스사이클 다음 epic의 기반** — 계약/배포/마감/정산/채권 9개 메뉴가 services row를 FK로 참조하게 됨. Folio가 source여야 통합 가능
4. **list-variants 신아키텍처 추가 ROI** — 4번째 도메인 (ai-work / cohort / backup / services) 추가로 패턴 안정성 검증

### 기각된 대안
- **B**: Sheets sync는 sync drift / 양방향 충돌 / Sheets schema 변경 발생 시 디버깅 비용 큼. services는 다른 메뉴(계약/배포)의 FK 기준이라 Folio가 source여야 정합성 유지
- **Z**: 본인 필터 / 검색 / 조직 권한 메뉴 연동 가치 모두 미실현. 사이드바 placeholder 잔존

## 검색 + 필터 UX

- **검색**: 상단 input — 대학명·서비스명 ilike. typing → 300ms debounce → 서버 query
- **필터 드롭다운**: 카테고리 (multi) + 지역 (multi) + 대학구분 (single) + 접수구분 (single) + 단독여부 (single)
- **본인 필터 토글**: "내 담당 서비스만" 체크박스 — operator_email = me.email OR developer_email = me.email
- **정렬**: 작성마감 임박순 (기본) / service_id / 등록 최근순
- **페이지네이션**: 12개씩 (ai-insight 패턴) 또는 30개씩 (table view라 밀도 ↑)

## Import 흐름 (대안 A 핵심)

### 1차: Google Sheets → CSV → 일회성 적재
1. 사용자가 Google Sheets에서 "다운로드 → CSV" → 같은 폴더(`.claude/memory/analysis/`)에 저장
2. 스크립트 `scripts/services-import.mjs`:
   - CSV 파싱
   - 로컬 Excel(`서비스리스트.xlsx`)을 매핑 dict로 로드 (`service_id → category`)
   - 카테고리 보강 (시트에 없으면 Excel 매칭)
   - operators 테이블 name→email 매칭 (best-effort)
   - service_id UNIQUE upsert (`ignoreDuplicates: false`)
3. import 결과: inserted N / updated M / matched_operators K / unmatched_operators L 로그

### 2차: 운영부가 신규 서비스 Folio에서 추가
- 등록 폼 → service_id 수동 입력 또는 자동 부여? — **수동 입력 권장** (외부 PIMS에서 부여한 ID 보존). 단 PIMS 미경유 신규는 Folio 자체 generated.
- service_id 충돌 시 zod 단계에서 거부

## Out of Scope

- **양방향 sync** — Google Sheets와 Folio가 자동 동기화 안 함 (대안 B 기각)
- **카테고리 자동 추론** — 시트에 없으면 등록 폼에서 운영자가 select로 직접 입력 (Excel 매칭 1회 보강만)
- **대학 마스터 도메인** — university_name은 text. 별도 epic으로 대학 도메인 신설 시 FK 마이그레이션
- **시즌 명시 컬럼** — 작성마감 연도로 추정. 별도 season 컬럼 추가는 후속 epic
- **계약/배포/마감/정산 메뉴와의 FK 연동** — 본 epic은 services 자체만. 후속 epic에서 FK 도입
- **알림 (마감 임박 7일 전 메일)** — follow-up
- **CSV export** — Folio → Sheets 역방향. 후속
- **gin to_tsvector 검색** — 1차는 ilike. 응답 > 500ms 측정 시 follow-up
- **운영자 매칭 실패 행의 자동 정정 UI** — 1차는 text fallback + 로그. 후속에서 운영자가 admin UI로 매칭 정정
