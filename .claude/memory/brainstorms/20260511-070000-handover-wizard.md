# Brainstorm: 인수인계 워크플로 (handover wizard + 메일 + PDF)

작성: 2026-05-11 / 사용자: 송영석

PR #37 머지 후 사용자가 다른 우선순위로 전환했으나, brainstorm 단계 결정은 인계 보존.

## 의도

- **산출물**: 3-step wizard (서비스 선택 → 인수자 선택 → 확인 → 완료) + 메일 자동 발송 + HTML→PDF 첨부
- **사용자**: 운영자가 휴가/이직/팀 이동 시 본인 담당 서비스를 다른 운영자에게 인계
- **트리거**: sidebar `handover` slug 있지만 현재 fallback. 운영자 다이어리 마지막 조각

## 사용자 합의 결정 (이번 세션)

1. **서비스 데이터**: 새 `services` 테이블 (DB) — `/dashboard/services`도 재사용 + 운영 이력 추적, 시드는 대학 담당 서비스
2. **PDF 생성**: **playwright** (이미 e2e용 설치됨, 재활용). `page.pdf()` API 사용

## 미결정 (다음 세션 첫 질문)

- 인계 내용 구조 — 자유 텍스트 1 필드 vs 항목별 체크리스트(현재 상태·미해결·연락처·자료) vs markdown
- 메일 발송 — Resend 권장 (Supabase invite는 신규 가입용이라 부적합) vs Gmail SMTP 재활용
- `handovers` 테이블 스키마: service_id / from_email / to_email / content(jsonb?) / status / sent_at / pdf_url
- PDF 저장소: Supabase Storage `handover-pdfs` bucket

## 분할 (예상)

- **PR-1**: services 테이블 + 시드 + listServices (RLS)
- **PR-2**: handovers 테이블 + 3-step wizard UI + draft 저장
- **PR-3**: 메일 발송 + playwright PDF 생성 + 첨부
- **PR-4**: 활동 로그 + 통계

## 사전 작업 (사용자)

- Supabase Storage bucket 'handover-pdfs' 생성 (RLS 정책 — 본인 인계 본인 download)
- Resend API key 발급 (메일 발송 시 사용)
