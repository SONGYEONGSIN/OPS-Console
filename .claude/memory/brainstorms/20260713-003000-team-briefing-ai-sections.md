# Brainstorm: 팀 보고 브리핑 — 구조 개편 + AI 활용 섹션 추가

## 의도

- **산출물**: `team-briefing` 자동화 잡의 Teams 메시지를 시안 B(차주 업무 중심형)로 개편하고, '내 AI 작업'(ai_work)·'TIP 공유'(ai_tips) 섹션 추가
- **사용자**: 운영부 전원 (매주 금요일 Teams 공지방 수신)
- **트리거**: 기존 브리핑이 계약/일정/마감만 다뤄 AI 활용 실적·공유가 보이지 않음 — AI 활용 문화 확산 목적
- **성공 기준**: 금요일 발송 메시지에 3개 대섹션(차주 팀 업무 / 계약현황 / AI 활용) + AI 작업 건수·절감시간·TIP 신규 목록이 정확히 집계되어 표시

## 제약

- **기술**: Teams 메시지 HTML 제약 (b/br/nbsp 수준) — 기존 빌더 스타일 유지. cron 실행이라 admin client 직접 조회
- **비즈니스**: ai_work의 author는 email만 저장 — operators 테이블로 이름 변환 필요 (email 노출 금지)
- **코드베이스**: 순수 빌더(team-briefing-build.ts) + 잡(team-briefing.ts) 분리 구조 유지, 빌더는 단위 테스트 필수(TDD). 잡 테스트의 chain mock 형태(select/gte/order)에 맞는 쿼리 shape 사용

## 추천 + 근거

**채택: 시안 B (차주 업무 중심형)** — 사용자 선택. 메시지 최종 구조:

```
[팀 보고 브리핑] 2026-07-13 (금)
■ 차주 팀 업무 (07-14 ~ 07-18)
 · 일정 — 유형별 그룹 (기존)
 · 서비스 마감 (7일 내 · 총 N건) — 날짜별 그룹 (기존)
■ 계약현황 (누적) — 시트별 + 합계 (기존)
■ AI 활용 (최근 7일)
 · 내 AI 작업 N건 · 절감 Hh — 목록 최대 5건 + "외 N건"
 · TIP 공유 (신규 n · 누적 m) — 목록 최대 5건
```

**기각 A**: 요약 헤더는 섹션이 3개뿐이라 중복 정보. 추후 섹션이 늘면 재검토 가치 있음.
