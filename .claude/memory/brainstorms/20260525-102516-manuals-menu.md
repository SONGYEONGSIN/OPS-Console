# Brainstorm: 운영 매뉴얼 메뉴 — /dashboard/manuals

생성 2026-05-25T10:25:16+09:00 · 주제: SharePoint 운영부/05. 매뉴얼 폴더(89 items)를 OPS-Console 메뉴로 노출

## 의도

- **무엇을**: `/dashboard/manuals` 메뉴 신설. SharePoint 메인 드라이브의 `운영부/05. 매뉴얼` 폴더 children(파일·하위 폴더)을 ListPattern + Inspector 구조로 노출. 행 클릭 시 SharePoint 웹으로 새 탭 이동(기존 권한 그대로). 하위 폴더는 query param으로 드릴다운.
- **누가**:
  - 운영부 전체(admin/member/viewer)가 read-only로 매뉴얼 검색·열람
  - 시점: 새 운영자 온보딩, 업무 중 절차 확인, 매뉴얼 어디 있는지 모를 때
  - 대체 행동: 현재는 SharePoint 직접 진입 → 운영부 → 05. 매뉴얼 폴더 클릭(3-hop)
- **왜 지금**: 매뉴얼 65+ 파일이 접두사 분류(A~I)로 정리돼 있지만 SharePoint UI는 단순 알파벳 정렬만 — 카테고리 구분이 시각적으로 안 보임. OPS-Console에서 한 화면에 카테고리별로 보이면 신규 운영자 진입 비용 ↓. 메뉴 미루면 매뉴얼 발견성 낮은 채로 운영부 신규 채용 시즌 진입.
- **성공**:
  - 메뉴 진입 1번 클릭 → A~I 그룹 + 폴더 4개 표시
  - 행 클릭 → SharePoint 새 탭(기존 세션 인증) — 다운로드/편집은 SharePoint UI에 위임
  - 하위 폴더 클릭 → 그 폴더 children 표시 + 상위 이동 가능
  - 검색어 입력 → 제목 부분 매칭(ListPattern 기본 검색)

## 제약

**기술**:
- Microsoft Graph: `GET /drives/{driveId}/items/{itemId}/children` (path 기반 가능). `SHAREPOINT_DRIVE_ID` 메인 드라이브 + 신규 `SHAREPOINT_MANUAL_ITEM_ID=01TGOQVTUKW6DM6CB3T5DIJOFQFXSGUV6F`
- Graph token 401 자동 갱신 패턴은 PR #207에서 추가됨. 매뉴얼 listChildren도 동일 패턴 적용 (forceRefresh + 1회 retry)
- 권한: `Files.Read.All` 또는 `Sites.Read.All` (이미 contracts/documents에서 사용 중)
- HARD-GATE 등급: 영향 파일 ~12개 → **간략 설계** (`rules/git.md`)

**비즈니스**:
- 매뉴얼 편집은 SharePoint에서 직접 — OPS-Console 측은 read-only
- 다운로드는 SharePoint webUrl이 처리 (별도 다운로드 API 호출 X)
- 운영부 전체 접근 — 외부 공유 매뉴얼이 있을 수 있지만 SharePoint RLS가 일차 가드

**코드베이스**:
- list-variants 아키텍처(open/closed) 재사용 — variant 추가 비용 1 폴더 + registry 1줄
- Inspector EditForm 불필요(read-only). View만 작성
- ListPattern 기본 검색·필터 사용 — 접두사 그룹은 Table.tsx에서 자체 그룹 헤더 렌더링

## 추천 + 근거

**추천**: **대안 A — 접두사 그룹 + ListPattern + Inspector**

**선택 근거**:
1. 다른 메뉴(서비스/계약/사고/공지/개선요청 등 16 variant)와 시각·인터랙션 일관 — 운영부가 학습 비용 0으로 사용
2. ListPattern의 검색·필터·페이지네이션·Inspector 인프라 그대로 재사용 — 신규 컴포넌트 코드 ↓
3. 향후 SharePoint 폴더 기반 메뉴 추가(예: 공문/문서 폴더) 시 동일 variant 재사용 가능

**기각된 대안 B**: 카드 그리드는 매뉴얼 자체 특성에 잘 맞지만, **다른 16 variant와 시각적으로 이질** → 운영부가 매뉴얼만 다른 UX로 학습해야 함. 디자인 토큰(`design-tokens.ts`) 외 신규 컴포넌트 추가 비용도 있음. **전환 가치 있는 시점**: 매뉴얼 외 비슷한 "리소스 폴더" 메뉴가 3+개 추가돼 카드 패턴 자체가 시스템에 자리잡을 때.

**기각된 대안 Z**: 운영부 신규 채용 시즌(분기별) 진입 비용을 매번 SharePoint 네비게이션 학습으로 지불. 메뉴 신설은 6시간 1회 비용으로 영구 해소.
