# Brainstorm: 실시간 현황(/dashboard) 전면 재설계 — 운영부 상황실

## 의도

- **산출물**: `/dashboard` 실시간 현황 페이지를 "운영부 상황실(Command Center)" 컨셉으로 재설계. 4영역 — ① 상단 커맨드바(날짜+KST 실시간 시계 + 게이트웨이 압축 LED + 백그라운드 로그 가로 티커) ② 서비스 라이프사이클 4카드(오픈예정→진행중→마감→전형료정산) ③ '처리 필요(Attention)' 보드 ④ 전 메뉴 소스 통합 테이블(기본 테이블/칩).
- **사용자**: 운영부 운영자(팀장/팀원)가 출근 직후·수시로 접속해 "지금 상황 + 오늘 처리할 것"을 한눈에 파악. 현재 대체행동 = 흩어진 KPI/로그/게이트웨이를 좌우로 훑고, 개별 메뉴를 따로 돌아다님.
- **트리거(왜 지금)**: 현 레이아웃이 (a) 시선 분산 (b) 우측 컬럼(게이트웨이/로그)이 본문 폭을 잠식 (c) 테이블이 일부 메뉴만 표시 → "상황실"로서 기능 부족. 사용자가 명시적으로 파격 재설계 요청.
- **성공 기준(측정가능)**: 우측 컬럼 제거로 본문 전폭 확보 / 게이트웨이·로그 상단 이동 완료 / 라이프사이클 4카드 동작 / Attention 보드가 심각도 신호화 / 테이블이 표시 가능한 전 메뉴 소스 + 기본 테이블·칩 적용 / 각 단계 PR이 독립 동작·검증.

## 제약

- **기술**: 기존 `_components/live/*` 컴포넌트 세트 재활용/대체. 실시간 시계·티커·LED 팝오버는 client("use client") 경계. 로그는 ConsoleStream/worklog 데이터 재사용. 테이블은 표준 ListPattern/기본 테이블 + ScopeChips로 교체(커스텀 LiveTable 폐기). 디자인 토큰 준수(하드코딩 색 금지, design.md). dashboard/page.tsx 데이터 조립을 전 메뉴로 확장.
- **비즈니스**: 전형료 정산은 카드 셸만(백엔드 후속). **단계별 PR**(운영 중 페이지라 점진 교체로 회귀 최소화). 디자인 취향 결정은 사용자.
- **코드베이스**: list-variants/ListPattern·ScopeChips 표준 재사용(중복 UI 추출 규칙). 20+ 파일 영향 가능 → HARD-GATE 전체설계 등급. RSC 직렬화 호환(인스펙터 variant 패턴).

|------|------------------|------------------------|-------------------|--------------|

| 브랜드 적합 | 보통 | **최상**(전 제품이 에디토리얼 톤) | 보통(미션컨트롤) | - |

### 파격 아이디어 풀 (대안에서 차용)
- **헤드라인 존**: 가장 시급한 1건(긴급 사고 / 오늘 마감 N건)을 1면 톱기사처럼 크게. "오늘 뭐부터?"를 즉답. (B)
- **기상도 게이트웨이**: 7개 LED를 "오늘의 시스템 날씨"(전부 green=맑음, 지연=흐림)로 은유. (B)
- **오늘의 타임라인**: 라이프사이클/마감을 '오늘 09:00→now→18:00' 가로 시간축에 점으로. (C/B)
- **심각도 부유(float)**: Attention 보드에서 적색 항목이 위로 떠오르고 pulse. (C)
- **로그 시세판 티커**: 백그라운드 로그를 가로 크롤(주식 시세판). (A)

## 추천 + 근거

**추천: A(3단 커맨드센터)를 골격으로 + B의 '마스트헤드·헤드라인' + C-lite '심각도 부유'를 주입한 하이브리드.**

- **근거**: 사용자가 A(상황실)·단계별 PR을 이미 채택 → A를 안전한 뼈대로. 그러나 "파격" 요구는 레이아웃 재배치만으론 부족 → **B의 헤드라인 존**(오늘 톱 이슈를 1면처럼)과 **기상도 게이트웨이**가 에디토리얼 브랜드와 정확히 맞고 "상황실다움"을 만든다. **C의 심각도 부유**는 Attention 보드에 국소 적용하면 저위험으로 "반응하는 대시보드" 감각을 준다.
- **기각 — B 단독**: 신문 은유를 페이지 전체에 강제하면 데이터 밀도/가독성 저하 위험. 톱 영역에만 차용.
- **기각 — C 단독**: 전면 적응형 재배치는 복잡·회귀 위험 높고 단계별 PR과 상충. 심각도 부유만 국소 차용.
- **기각 — Z**: 사용자가 명시적으로 재설계 요청, 현 문제(폭 잠식·분산·부분 테이블) 지속.
