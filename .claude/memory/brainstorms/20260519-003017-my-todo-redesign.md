# Brainstorm: /dashboard/my-todo 재설계 — 2 탭(원서접수 Weekly Planner + 프로젝트 Gantt)

작성: 2026-05-19 / 사용자: 송영석 / 트리거: 사용자 직접 요청 + 엑셀 Weekly Planner / Project Management mock 2장 제시

## 의도

- **산출물**: `/dashboard/my-todo` 페이지를 2 탭 구조로 재설계
  - **Tab 1 (원서접수)**: Weekly Planner — 주(week) 단위 그리드 + 본인 todos 테이블, 카테고리/우선순위/상태 필터, 행별 진행률 % 막대
  - **Tab 2 (프로젝트)**: Project Management — 프로젝트(부모) + 1단계 sub-task + 담당자/시작/마감/우선순위/진행률 + Gantt 차트(월 timeline 막대)
- **사용자**:
  - admin/member 본인: 개인 일정 + 본인 프로젝트 관리
  - viewer: menu 권한에 따라 진입 가능, read-only
  - 데이터 소유: **본인 only** (RLS — created_by_email = auth.email())
- **트리거**: 사용자 직접 요청. 기존 services-based planner 흐름 완전 폐기 확정
- **성공 기준**:
  1. URL 기반 탭 토글 (`?tab=weekly|project`, default=weekly)
  2. Tab1 본인 todos CRUD + 주별 그리드 표시 + 카테고리/우선순위/상태 필터
  3. Tab2 본인 projects + sub-tasks CRUD + Gantt 시각화 (월별 timeline 막대)
  4. RLS — 본인 created_by 행만
  5. lint/typecheck/test 통과 + 기존 my-todo e2e 영향 없거나 deprecated 처리

## 제약

- **기술**:
  - Next.js 16 App Router — page는 RSC, WeeklyView / GanttChart는 client
  - Supabase RLS auth.uid() 기반 본인 행만 SELECT/UPDATE/DELETE
  - list-variants 아키텍처 활용 — `weekly-todo` / `project` / `project-task` 3 신규 variant
  - Gantt 차트는 라이브러리 없이 Tailwind CSS grid + 토큰으로 직접 구현 (vermilion/sage/indigo)
  - KST timezone 일관 (Vitest 이미 TZ=Asia/Seoul)
- **비즈니스**:
  - v1 sub-task 깊이 1단계만 (parent project + children tasks)
  - 카테고리는 자유 입력 string (admin 마스터는 후속)
  - Gantt 단위는 일(day), 주(week), 월(month) 중 일 단위 v1 (mock 따라)
- **코드베이스**:
  - 기존 `MyTodoLayout.tsx`, services-based planner 코드 완전 deprecate (삭제)
  - `todos.source_service_id` FK 제거 (마이그)
  - 새 컬럼: `todos.category`, `todos.priority`, `todos.status`, `todos.progress` (모두 nullable로 기존 데이터 무손상)
  - 신규 도메인: `features/projects/` (schemas/queries/actions/__tests__)

## 추천 + 근거

**선택: 대안 A — todos 확장 + projects 신설**

- 기존 todos 활용 — 신규 컬럼 nullable로 추가, 기존 데이터 무손상
- projects/project_tasks는 1:N 관계로 깔끔 분리 (v1 sub-task 깊이 1단계)
- Gantt 차트는 별도 시각 컴포넌트, projects 데이터를 timeline으로 단순 변환
- list-variants 패턴 재사용 — `weekly-todo` / `project` / `project-task` 3 variant 추가

**기각 — B**: weekly_todos와 todos 두 테이블 분리하면 카테고리/우선순위 같은 공통 필드를 두 번 정의 + UI도 중복. 도메인 분리 가치 < 중복 비용
**기각 — C**: 같은 테이블에 weekly/project root/sub-task 섞이면 type 분기 로직이 곳곳에 산재. 단일 type=weekly 케이스가 가장 흔한데 type 분기 cost가 매번 발생. 안티-패턴
**기각 — Z**: 사용자 명시 요청 (엑셀 mock 2장)
